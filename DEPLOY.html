Rent vs Buy BETA DEPLOY

AWS deployment guide for rentbuysellapp.com — March 2026

1. Prerequisites Checklist

  • AWS account with admin (or IAM with EC2, S3, CloudFront, Route 53, ACM, SES)
  • AWS CLI v2 installed: aws --version ≥ 2.x
  • AWS CLI configured: aws configure (access key + region us-east-1)
  • Node 18+ locally: node --version
  • Python 3.12+ locally: python --version
  • Domain rentbuysellapp.com in Route 53 (hosted zone exists)
  • SSH key pair for EC2 created in AWS Console

2. Build & Package Locally

Frontend

cd web
NEXT_PUBLIC_API_URL=https://api.rentbuysellapp.com npm run build
# Output: web/out/ (static files, ~2MB)

Backend dependencies

# From api/ directory
pip install numpy scipy pandas fastapi uvicorn requests boto3
Do not run npm run build while npm run dev is running. It overwrites .next/ and breaks CSS. Stop the dev server first.

3. Frontend: S3 + CloudFront

Step 1: Create S3 Bucket

  1. AWS Console → S3 → Create bucket
  2. Bucket name: rentbuysellapp.com
  3. Region: us-east-1
  4. Block all public access: ON (CloudFront serves, not S3)
  5. Leave everything else default → Create

Step 2: Upload static files

aws s3 sync web/out/ s3://rentbuysellapp.com --delete

Step 3: Create CloudFront Distribution

  1. CloudFront → Create distribution
  2. Origin domain: select your S3 bucket (rentbuysellapp.com.s3.amazonaws.com)
  3. Origin access: Origin Access Control (OAC) — click "Create new OAC", type S3, Sign requests → Create
  4. Default root object: index.html
  5. Viewer protocol policy: Redirect HTTP to HTTPS
  6. Cache policy: CachingOptimized (managed)
  7. Price class: Use all edge locations (or North America + Europe to save)
  8. Alternate domain names (CNAMEs): rentbuysellapp.com and www.rentbuysellapp.com
  9. Custom SSL certificate: select your ACM cert (see Section 5 — must be in us-east-1)
  10. Create distribution

Step 4: Custom error response (SPA routing)

  1. In the new distribution → Error pages tab
  2. Create custom error response: HTTP error code 403 → Response page path /index.html → HTTP response code 200
  3. Create another for 404 → /index.html → 200

Step 5: Update S3 bucket policy

After creating the distribution, CloudFront shows a banner: "S3 bucket policy needs to be updated." Click Copy policy, go to your S3 bucket → Permissions → Bucket policy → paste → Save.

OAC vs OAI: OAI (Origin Access Identity) is deprecated since 2023. Always use OAC for new distributions.

4. Backend: EC2 (Recommended for Beta)

Step 1: Launch instance

  1. EC2 → Launch instance
  2. Name: rentbuy-api
  3. AMI: Amazon Linux 2023
  4. Instance type: t3.small (2 GB RAM — needed for market.db + numpy)
  5. Key pair: select yours
  6. Security group: create new:
    • SSH (22) from My IP
    • Custom TCP (8000) from the ALB security group (or anywhere for initial testing)
  7. Storage: 20 GB gp3
  8. Launch

Step 2: Install dependencies

ssh -i your-key.pem ec2-user@<public-ip>

# Install Python 3.12
sudo dnf install python3.12 python3.12-pip git -y

# Clone repo
git clone https://github.com/YOUR_USER/mortgage.git
cd mortgage/api

# Install Python deps
python3.12 -m pip install numpy scipy pandas fastapi uvicorn requests boto3

# Upload your data (from local machine)
# Option A: scp
scp -i your-key.pem api/data/market.db ec2-user@<ip>:~/mortgage/api/data/

# Option B: download fresh
python3.12 refresh_data.py

Step 3: Create systemd service

sudo tee /etc/systemd/system/rentbuy.service <<'EOF'
[Unit]
Description=Rent vs Buy API
After=network.target

[Service]
Type=simple
User=ec2-user
WorkingDirectory=/home/ec2-user/mortgage/api
ExecStart=/usr/bin/python3.12 -m uvicorn api:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=5
Environment=MORTGAGE_DB_PATH=/home/ec2-user/mortgage/api/data/market.db

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable rentbuy
sudo systemctl start rentbuy

# Verify
curl http://localhost:8000/health
# Should return: {"status":"ok"}

Step 4: Application Load Balancer

  1. EC2 → Load Balancers → Create → Application Load Balancer
  2. Name: rentbuy-alb
  3. Scheme: Internet-facing
  4. Listeners: HTTP (80) and HTTPS (443)
  5. Availability zones: select at least 2
  6. Security group: allow 80 and 443 from anywhere
  7. Target group: create new, HTTP port 8000, health check path /health
  8. Register your EC2 instance
  9. HTTPS listener: select ACM certificate for api.rentbuysellapp.com
  10. HTTP listener: add rule to redirect to HTTPS

5. SSL Certificates (ACM)

  1. AWS Console → Certificate Manager → Request certificate
  2. Domain names: rentbuysellapp.com and *.rentbuysellapp.com
  3. Validation: DNS validation
  4. Click Create records in Route 53 (one-click — adds CNAME records automatically)
  5. Wait 2–10 minutes for status to change to "Issued"
Region matters! CloudFront requires certs in us-east-1. If your ALB is in a different region, you need a second certificate there. Request the cert in us-east-1 first.

6. Route 53 DNS Records

Go to Route 53 → Hosted zones → rentbuysellapp.com → Create records:

7. AWS SES Setup (Alert Emails)

  1. SES Console → Verified identities → Create identity
  2. Identity type: Domain
  3. Domain: rentbuysellapp.com
  4. Click Publish DNS records → automatically adds DKIM records to Route 53
  5. Wait for verification (usually <5 minutes)
  6. SES starts in sandbox mode — can only send to verified emails
  7. For beta, verify your personal email as a recipient
  8. For GA, request production access: SES → Account dashboard → Request production access

IAM permissions for EC2

Create an IAM role for your EC2 instance with this policy:

{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Action": ["ses:SendEmail", "ses:SendRawEmail"],
    "Resource": "arn:aws:ses:us-east-1:*:identity/rentbuysellapp.com"
  }]
}

Attach the role to your EC2 instance: EC2 → Instance → Actions → Security → Modify IAM role.

8. Feedback Email Routing

The UI links to feedback@rentbuysellapp.com. You need to route incoming mail to your real inbox. Two options:

Option A: SES Email Receiving + S3 + Lambda (AWS-native)

  1. SES → Email receiving → Create rule set → Create rule
  2. Recipients: feedback@rentbuysellapp.com
  3. Actions: Store in S3 bucket (creates a new one) + SNS notification to your email
  4. Add MX record in Route 53: rentbuysellapp.com MX 10 inbound-smtp.us-east-1.amazonaws.com
SES email receiving only works in us-east-1, us-west-2, and eu-west-1. Make sure you set it up in one of those regions.

Option B: Email forwarding service (simpler)

Use a free service like ImprovMX or Forward Email:

  1. Sign up, add rentbuysellapp.com
  2. Set up forwarding: feedback@rentbuysellapp.com → your-real-email@gmail.com
  3. Add their MX records to Route 53 (they provide exact values)
  4. Takes 5 minutes, no Lambda/S3 needed

10. Data Refresh Cron

# SSH to EC2, then:
crontab -e

# Add this line (runs every Sunday at 6am UTC):
0 6 * * 0 cd /home/ec2-user/mortgage/api && /usr/bin/python3.12 refresh_data.py >> /var/log/rentbuy-refresh.log 2>&1

For initial setup, run manually with --skip-notifications:

cd ~/mortgage/api && python3.12 refresh_data.py --skip-notifications

11. CORS for Production

In api/api.py, change:

# Current (dev):
allow_origins=["*"]

# Production:
allow_origins=[
    "https://rentbuysellapp.com",
    "https://www.rentbuysellapp.com",
]
For beta: you can leave * temporarily. Tighten before GA.

13. Post-Deploy Verification

  • Open https://rentbuysellapp.com — page loads with SSL padlock
  • Open https://api.rentbuysellapp.com/health — returns {"status":"ok"}
  • Run a simulation from the UI — results appear
  • Open DevTools Network tab — API calls go to api.rentbuysellapp.com
  • Test on mobile phone (same WiFi or cellular)
  • Reload the page — second load is fast (CloudFront cached)
  • Check https://rentbuysellapp.com/robots.txt loads
  • Check https://rentbuysellapp.com/sitemap.xml loads
  • Send test email to feedback@rentbuysellapp.com from a personal email
  • Submit Google Search Console: search.google.com/search-console → add property → submit sitemap
